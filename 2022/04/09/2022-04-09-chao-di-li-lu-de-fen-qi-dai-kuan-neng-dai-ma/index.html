超低利率的分期贷款能贷吗？

01. 先说结论

90%的情况下是不能。

虽然贷款年利率3.6%，你的投资回报能到4%，但是你每年还是会亏掉1.66%。

02. 得出结论的原因

90%是基于如下的假设（其中的百分比数据均为假设）：

  1. 贷款的理由：1% 贷款救命，99% 加杠杆投资

  2. 年投资回报率：90% 回报率低于6.7%，10% 回报率高于6.7%

    贷款救命自不必说，必须得贷，和钱无关，不在讨论范围内。

    加杠杆投资和投资回报率低同时满足的人约等于90%，而满足这两个条件，你的投资基本就是亏定了，所以90%的情况是不能。

选择6.7%的年投资回报率的原因是这个利率下，收支基本能平衡（贷款5年，3.6%年利率的情况下）。

具体计算过程如下：

假设我们打算分期贷款10w，银行给的超低年化利率是3.6%，贷款60期（60个月，5年）。

5年的利息总额是1.8w，每个月还1967元。

如果将这10w拿去投资，假设收益年化6.7%，

因为每个月还需要偿还本金和利息，所以5年的盈利是1.795w。

基本做到收支平衡。如果年收益率低于6.7%那就只能亏损了，如果能超过6.7%那当然还是能挣钱的。

然而实际上能做到4%的年化收益率在普通人中就算比较好的了，此时的5年盈利是0.97w。

最终亏损0.83w（1.8w - 0.97w），亏损率8.3%（0.83w ➗ 10w），平均每年约1.66%（6.8% ➗ 5）的亏损。

03. 仔细聊聊

很多奔着挣钱去的人都无法理解为什么会这样。

明明贷款利率3.6%是低于我的投资回报率4%的，为什么还亏钱了？

关键点在于本金不同，贷款利率3.6%是一直按照贷款额度10w来计算利息，而你的4%年投资回报率是扣除月供后的金额来计算利息，本金肯定远低于10w。

用同样的例子来说明，贷款10w，贷5年（60期），贷款年利率3.6%，年投资收益率4.0%。

收益计算公式如下：
$$
收益 = 投资收益 - 贷款支出
$$

5年的贷款支出：
$$
10w \times 3.6% \times 5 = 1.8w
$$
5年的投资收益为：
$$
第1期投资回报E_1 + 第2期投资回报E_2 + 第3期投资回报E_3 + … + 第60期投资回报E_{60} = 0.97w
$$
随着本金用于偿还贷款利息而越来越少，每一期的投资收益会越来越少，收益计算如下：
$$
E_1 + E_2 + … + E_{60} ≈ 0.97w
$$

$$
E_1 = 当前本金10w \times 4.0% \div 12 ≈ 333
$$

$$
E_2 = (偿还第1期利息前本金100’000 - 上期月供1967 + 上月投资收益333) \times 4.0% \div 12 ≈ 328
$$

$$
E_3 = (偿还第2期利息前本金98’366 - 上期月供1967 + 上月投资收益328) \times 4.0% \div 12 ≈ 322
$$

...

$$
E_{60} = (偿还第59期利息前本金-6301 - 上期月供1967 + 上月投资收益-14) \times 4.0% \div 12 ≈ -21
$$

最终便可以得出结论：

亏损0.83w（1.8w - 0.97w），亏损率8.3%（0.83w ➗ 10w），平均每年约1.66%（6.8% ➗ 5）的亏损。

回顾一下上面的过程，本质上是存在两个原因导致投资结果与预期不符：

  1. 产生收益的本金会逐月减少（每月需要还贷款）
  2. 用于计算贷款利息的贷款总额不会逐月减少（与按揭贷款不同，分期贷款的利息不会逐月减少，所以实际的贷款利率要比宣传的高得多）

04. 编程计算收益

一个简单的递归可以很快实现这种本金持续变化的情况，代码如下：

计算的代码片段（Java）

public class InterestCalculator &#123;
    private double loanAmount; // 贷款金额
    private double loanRate;  // 贷款年利率
    private double interestRateMonthly;  // 月收益率
    private int installmentNumber;  // 分期期数
    private double installmentPayment;  // 月供
    private double serviceCharge;  // 服务费

    public InterestCalculator(double loanAmount, double loanRate, double interestRateMonthly, int installmentNumber, double serviceCharge) &#123;
        this.loanAmount = loanAmount;
        this.loanRate = loanRate;
        this.interestRateMonthly = interestRateMonthly;
        this.installmentNumber = installmentNumber;
        this.serviceCharge = serviceCharge;
        this.installmentPayment = loanAmount / installmentNumber + loanAmount * loanRate / 12;
    &#125;

    public double calculateInterestIncome(double principal, int installmentIndex, double totalInterestIncome) &#123;
        if (installmentIndex > installmentNumber) &#123;
            return totalInterestIncome;
        &#125;
        System.out.println("第[" + installmentIndex + "]期：");
        System.out.println("本金：" + principal);
        System.out.println("月供：" + installmentPayment);
        double interestThisMonth = principal * interestRateMonthly;
        System.out.println("利息收入：" + interestThisMonth);
        double principalNextMonth = principal + interestThisMonth - installmentPayment;
        totalInterestIncome += interestThisMonth;
        return calculateInterestIncome(principalNextMonth, installmentIndex + 1, totalInterestIncome);
    &#125;

    public double calculateMargin(double principal, int installmentIndex, double totalInterestIncome) &#123;
        return calculateInterestIncome(principal, installmentIndex, totalInterestIncome) - serviceCharge - loanAmount * loanRate * installmentNumber / 12;
    &#125;
